The `esp_audio_stack` microphone platform exposes the capture stream owned by
the `esp_audio_stack` component as a normal ESPHome microphone.

When `processor_id` is configured and enabled on the stack, this microphone
exposes the post-processor stream. If the enabled processor is unavailable,
output is silence. Disabling the parent AEC switch is an explicit bypass that
publishes converted raw mic through this same entity; it does not create a
second microphone path.

```yaml
esp_audio_stack:
  id: audio_stack
  sample_rate: 48000
  output_sample_rate: 16000

microphone:
  - platform: esp_audio_stack
    id: clean_mic
    esp_audio_stack_id: audio_stack
```

## Configuration variables

- **esp_audio_stack_id** (**Required**, ID): The parent `esp_audio_stack` component.
- **id** (*Optional*, ID): Manually specify the microphone ID.
