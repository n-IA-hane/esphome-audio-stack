The `esp_aec` component wraps Espressif ESP-SR acoustic echo cancellation and
implements the audio-processor interface used by `esp_audio_stack`.

Use it when the device needs software echo cancellation but not the full AFE
pipeline.

```yaml
esp_aec:
  id: aec_processor

esp_audio_stack:
  id: audio_stack
  processor_id: aec_processor
```

The parent `esp_audio_stack` component owns the I2S bus and echo reference.

## Configuration variables

- **id** (*Optional*, ID): Manually specify the processor ID.
- **sample_rate** (*Optional*, int): ESP-SR AEC rate. Only `16000` is accepted
  and it is the default.
- **filter_length** (*Optional*, int): ESP-SR filter-length parameter, from `1`
  to `8`. Defaults to `4`; effective time coverage depends on the selected
  engine/frame shape.
- **mode** (*Optional*, string): AEC engine and profile. One of `sr_low_cost`, `sr_high_perf`, `fd_low_cost`, `fd_high_perf`, `voip_low_cost` and `voip_high_perf`. Defaults to `sr_low_cost`.

## Actions

### `esp_aec.set_mode`

Switches the AEC mode at runtime by rebuilding its ESP-SR handle. Stop the audio
path or otherwise ensure no real-time consumer is active first.

```yaml
on_...:
  - esp_aec.set_mode:
      id: aec_processor
      mode: fd_high_perf
```
